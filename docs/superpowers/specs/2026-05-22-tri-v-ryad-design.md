# Кати-Ряд — Дизайн-документ (MVP)

**Дата:** 2026-05-22
**Тип:** Веб-игра (match-3 с уникальной механикой)
**Цель:** MVP, который можно показать и в который реально играть. Без перегруза.

## 1. Идея и уникальная механика

Match-3 игра, где **игрок не двигает фишки по одной**, а наклоняет всё поле в одну из четырёх сторон (←↑→↓). Под действием "гравитации" все фишки катятся в эту сторону до упора. После остановки три и более одинаковых в ряд (горизонталь или вертикаль) исчезают, остальные снова катятся — получаются комбо-цепочки. Когда цепочка кончилась, на поле спавнятся новые фишки. Цель — выжить как можно дольше до того, как поле забьётся.

Аналог по ощущениям: 2048 + match-3. Такой игровой механики в массовых match-3 нет — отсюда уникальность.

## 2. Функциональные требования

### 2.1 Игровое поле

- Размер: 6×6 клеток.
- Каждая клетка содержит ровно одну фишку или пуста.
- 5 типов фишек, каждый со своей формой и цветом (форма дублирует цвет, чтобы игра работала для дальтоников).
- Поле визуально занимает центральную часть экрана, сверху — счёт, рекорд, кнопка "Заново".

### 2.2 Старт раунда

- Поле очищается.
- На поле спавнятся 12 случайных фишек в случайные клетки.
- Стартовая раскладка гарантированно **не содержит** готовых троек в ряд (если случайно сгенерировалось — генерация повторяется).
- Сбрасываются счёт и счётчик комбо.

### 2.3 Наклон поля (главное действие)

Триггеры: стрелка клавиатуры, WASD, свайп пальцем по полю, экранная кнопка-стрелка. Все триггеры вызывают одну функцию `tilt(direction)`, где direction ∈ {left, right, up, down}.

Алгоритм одного наклона:

1. **Сдвиг.** Все фишки по очереди "падают" в направлении наклона до тех пор, пока не упрутся в стенку или в другую фишку. Пустые клетки остаются позади.
2. **Проверка матча.** После остановки сдвига сканируем поле на наличие трёх и более одинаковых фишек подряд по горизонтали или вертикали.
3. **Если матчи есть:**
   - Все фишки в матчах помечаются на удаление.
   - Начисляются очки: `+10 × количество_фишек × множитель_комбо` (см. 2.5).
   - Помеченные фишки удаляются (анимация исчезновения).
   - Множитель комбо увеличивается на 1.
   - Возврат к шагу 1 (повторный сдвиг в том же направлении, новый поиск матчей).
4. **Если матчей нет** — цепочка завершилась, переходим к спавну (2.4).

Важно: если на шаге 1 в самом начале **ничего не сдвинулось** (поле уже прижато к этой стенке), то ход не считается — спавн не происходит, состояние не меняется. Это защищает от абуза "стучать в одну стенку, чтобы спавнить фишки".

### 2.4 Спавн новых фишек

После завершения цепочки матчей:

- Берётся список пустых клеток.
- В 3 случайных пустых клетки кладётся 3 новые фишки случайных типов.
- Если пустых клеток меньше 3 — кладётся столько, сколько помещается.
- Если пустых клеток 0 — спавн пропускается, но это не game over сам по себе (см. 2.6).

### 2.5 Очки и комбо

- Базово: каждая удалённая фишка даёт 10 очков.
- Множитель комбо: первая волна матчей в одном ходу × 1, вторая × 2, третья × 3 и т.д.
- Формула за одну волну: `+10 × количество_удалённых_фишек_в_этой_волне × текущий_множитель_комбо`.
- Счётчик комбо сбрасывается в начале каждого нового наклона (до первого матча = 1).
- Общий счёт раунда отображается сверху.
- Локальный рекорд хранится в localStorage и обновляется после game over.

### 2.6 Game Over

Условие конца игры: после спавна на шаге 2.4 **ни один из четырёх возможных наклонов не сдвинул бы ни одной фишки** (поле статично во всех направлениях). На практике это означает, что поле полностью заполнено и все фишки уже прижаты со всех сторон — то есть полностью забито.

Действия при Game Over:
- Показывается оверлей с финальным счётом и рекордом.
- Если счёт превысил рекорд — об этом сообщается ("Новый рекорд!").
- Кнопка "Заново" сбрасывает состояние и стартует новый раунд (см. 2.2).

### 2.7 Управление

- **Клавиатура:** стрелки `←↑→↓` или клавиши `A W D S`.
- **Сенсор:** свайп пальцем по полю (порог ~30px, направление по доминирующей оси).
- **Мышь / тач:** 4 экранные кнопки-стрелки по сторонам поля (всегда видимы).
- Все способы вызывают одну функцию `tilt(direction)`.
- Во время анимации (сдвиг / удаление / спавн) ввод блокируется.

## 3. Технический дизайн

### 3.1 Стек

- Чистый HTML + CSS + JavaScript (vanilla), **без сборки и без зависимостей**.
- Деплой: GitHub Pages — пуш в репозиторий, GitHub Pages раздаёт статику.
- Это совпадает со стилем коллег (Snow-bro, Balda, Snake-survivor) и минимизирует трение.

### 3.2 Структура файлов

```
tri_v_ryad/
├── index.html      — разметка: контейнер поля, табло, кнопки
├── style.css       — CSS Grid поля, стили фишек (форма+цвет), анимации
├── game.js         — состояние, логика, ввод, рендер
└── README.md       — описание, как играть, как задеплоить
```

Всё умещается в один файл `game.js`, разделённый на логические секции:
- Состояние (`state`)
- Логика поля (gravity, match detection, spawn)
- Рендер (синхронизация DOM с состоянием)
- Ввод (клавиатура, свайпы, кнопки)
- Игровой цикл (init, tilt, game over, restart)

### 3.3 Модель состояния

```js
state = {
  board: Array(6).fill().map(() => Array(6).fill(null)),
  //       board[row][col] = { type: 0..4 } | null
  score: 0,
  bestScore: 0,         // из localStorage
  comboMultiplier: 1,   // сбрасывается перед каждым наклоном
  isAnimating: false,   // блокирует ввод во время анимации
  isGameOver: false,
}
```

### 3.4 Рендер

- Поле — `display: grid` 6×6 в `<div id="board">`.
- Каждая фишка — `<div class="tile tile-{type}">`, спозиционирован абсолютно внутри своей grid-ячейки.
- Скольжение — CSS-transition по `transform: translate(...)`, длительность 200мс.
- Исчезновение — класс `removing`, который запускает CSS-keyframe-анимацию `scale 1→0` + `opacity 1→0`, длительность 200мс.
- Спавн — класс `spawning`, CSS-keyframe-анимация `scale 0→1` + `opacity 0→1`, длительность 200мс.
- Каждый шаг цепочки (сдвиг + матч + удаление) ждёт окончания предыдущей анимации, через `await delay(200)` или `await Promise<transitionend>`.

### 3.5 Гравитация (ключевой алгоритм)

Для направления `left`:
- Для каждой строки: пройти слева направо, "сжать" все non-null элементы влево, оставив null'ы справа.
- Реализация: `row = [...row.filter(c => c !== null), ...Array(N - filtered.length).fill(null)]`.

Для `right` — то же, но фишки прижимаются к концу.
Для `up` / `down` — то же по столбцам (можно реализовать через транспонирование или прямой проход).

Чтобы анимировать корректно, нужно вычислять **старое и новое положение каждой фишки**, не пересоздавая объекты. Например, каждая фишка имеет stable id, и при сдвиге мы строим map id → newRow,newCol, а в рендере анимируем CSS-transform от старой ячейки к новой.

### 3.6 Поиск матчей

После сдвига:
- Проходим все строки: ищем последовательности `>= 3` одинаковых подряд → помечаем их координаты.
- То же для столбцов.
- Объединяем множества координат (через `Set` с ключом `"r,c"`).
- Если множество непустое — это матч, удаляем эти координаты.

### 3.7 Проверка game over

После спавна:
- Для каждого из 4 направлений: симулировать сдвиг (на копии состояния) и проверить, изменилось ли поле.
- Если ни один из 4 наклонов ничего не меняет → game over.

(Альтернатива: проверять "есть ли вообще пустая клетка". Но симуляция надёжнее: возможна ситуация, когда есть пустые клетки, но фишки все упёрты, и наклон ничего не даст. Хотя на 6×6 такое маловероятно — наклон обычно что-то двигает, если есть пустые. Всё равно симуляция дешёвая, делаем её.)

### 3.8 Сохранение рекорда

- `localStorage.getItem('tri_v_ryad_best')` при загрузке.
- При Game Over: `localStorage.setItem('tri_v_ryad_best', Math.max(...))`.

### 3.9 Внешний вид (минимализм)

- Фон: один спокойный цвет.
- Поле: тёмный квадрат с лёгкой сеткой.
- Фишки: 5 разных форм (circle, square, triangle, rhombus, hexagon), реализованных через CSS (`border-radius`, `clip-path`), каждая со своим характерным цветом (различимые оттенки + форма для дальтонизма).
- Шрифт: системный sans-serif (или один Google Font, если нужен характер).
- Адаптив: поле квадратное, занимает `min(90vw, 90vh, 500px)`.

## 4. Что НЕ входит в MVP

Зафиксировано явно, чтобы не разрастаться:

- Бомбы, спец-фишки, бустеры
- Уровни, прогрессия, разблокировка
- Звук и музыка
- Онлайн-лидерборд, аккаунты
- Несколько тем оформления
- Туториал (вместо него — короткая подсказка управления на главном экране)

Всё это легко добавить в v2, если игра зайдёт.

## 5. Критерии готовности MVP

- [ ] Игру можно открыть на `localhost` или GitHub Pages и начать играть с первого экрана.
- [ ] Наклон в любом направлении (клавиатура/свайп/кнопка) работает и анимируется плавно.
- [ ] Матчи находятся корректно, очки начисляются, комбо-множитель работает.
- [ ] Новые фишки спавнятся каждый ход.
- [ ] Корректно срабатывает game over, показывается оверлей, кнопка "Заново" сбрасывает игру.
- [ ] Рекорд сохраняется между сессиями.
- [ ] Работает на десктопе (стрелки) и на телефоне (свайп).
- [ ] README объясняет как играть и как задеплоить на GitHub Pages.
