# Кати-Ряд — "Вау"-механики (v2 features)

**Дата:** 2026-05-22
**Тип:** Расширение MVP
**Цель:** Добавить дофамин-моменты — видимые комбо, спец-фишки от длинных матчей, screen shake. Базовая механика и баланс не меняются.

## 1. Что добавляем

Четыре скоординированных фичи поверх готового MVP:

1. **Видимые комбо** — на 2-й, 3-й и т.д. волне каскада посреди поля всплывает плашка "КОМБО ×2!" / "×3!".
2. **Match-4 = бомба** — собрал 4 фишки в ряд → на их месте остаётся бомба (фишка с обводкой). При следующем матче с её участием взрывается 3×3.
3. **Match-5+ = радужная фишка** — собрал 5+ в ряд → остаётся радуга. В будущем матче считается любым цветом и убирает все фишки этого цвета с поля.
4. **Screen shake** — поле слегка трясётся при значимых событиях (комбо ≥2, взрыв бомбы, активация радуги).

## 2. Функциональные требования

### 2.1 Модель данных фишки

Текущая фишка: `{ id, type }` где type ∈ {0, 1, 2, 3}.

Новая фишка: `{ id, type, special? }` где:

- `special` отсутствует или undefined → обычная фишка
- `special === 'bomb'` → бомба, у неё есть свой цвет (`type ∈ {0..3}`), при удалении взрывается 3×3
- `special === 'rainbow'` → радуга, у неё `type === -1` (нет цвета), вайлдкард при match-детекции

### 2.2 Детекция матчей с радугой

`findMatches` теперь возвращает массив матч-объектов:

```
{
  tiles: Set<"r,c">,    // все координаты этого матча
  length: number,        // длина run'а (3, 4, 5+)
  color: number,         // тип цвета (anchor color)
  center: "r,c",         // координата центральной позиции в run'е (для спавна спец-фишки)
  hasRainbow: boolean,   // содержит ли run радужные фишки
  hasBomb: boolean,      // содержит ли run бомбы
}
```

`center` вычисляется как координата позиции `Math.floor(length / 2)` от начала run'а в направлении сканирования (для горизонтальных runs — `start + floor(length/2)` по столбцу, для вертикальных — по строке).

Алгоритм поиска runs в линии (горизонталь или вертикаль):

1. Пройти по линии, сегментируя на серии non-null фишек.
2. В каждом сегменте определить "anchor color" — тип первой не-радужной фишки.
3. Серия валидна как матч, если:
   - длина ≥ 3
   - в ней есть хотя бы 1 не-радужная фишка (только радуги — не матч)
   - все не-радужные фишки одного типа (anchor color)
4. Если в серии встречается фишка другого типа — серия прерывается, начинается новый сегмент с этой фишки.

**Пример:** линия `[A, A, R, R, B, B]` → один валидный run `[A, A, R, R]` длины 4 цвета A. Дальше `[B, B]` длины 2 — не матч.

**Пример:** `[R, A, A, A, R, B]` → run `[R, A, A, A, R]` длины 5 цвета A (рассматривается жадно слева направо — обе радуги уходят с A, B остаётся одинокой).

Внутри `findMatches` сначала собираются все горизонтальные runs ≥ 3, потом все вертикальные. Если позиция входит в два run'а (L-shape) — она помечается в каждом, но при объединении флаги `hasRainbow`/`hasBomb` могут стоять в обоих матч-объектах. Это нормально — обработчик применит эффекты по разу для каждой фишки.

### 2.3 Создание спец-фишек после матча

После того как матч найден и фишки помечены на удаление, перед самым удалением:

- Если `length === 4` И в этом матче ещё **не было** активации (т.е. не сработала бомба/радуга в нём) → на месте **центральной позиции** (`Math.floor(length / 2)`-й по счёту в линии run'а) создаётся **бомба** цвета `color`. Эта фишка не удаляется, остальные 3 удаляются.
- Если `length >= 5` → на центральной позиции создаётся **радуга** (`type: -1, special: 'rainbow'`). Остальные удаляются.
- Если `length === 3` → обычное удаление, спец-фишек не создаётся.

**Важно:** если матч "активный" (см. 2.4), новые спец-фишки **не создаются** — это предотвращает бесконечные цепочки бомба→бомба→радуга.

### 2.4 Активация спец-фишек

Когда фишка со `special` оказывается в множестве удаления текущей волны — активируется эффект:

**Бомба (`special: 'bomb'`)**: к удалению добавляются все 8 соседних клеток (3×3 вокруг бомбы минус сама). Если в этих клетках есть другие бомбы — они тоже активируются (BFS до сходимости). Если есть радуги — они **не активируются** (радуга активируется только когда сама в матче, не от соседних взрывов).

**Радуга (`special: 'rainbow'`)**: к удалению добавляются все фишки на доске того же `color`, что и матч, в котором участвует радуга. Их обычные тайлы, не спец. Если среди них есть бомбы — те активируются (запускают свои 3×3).

Цикл расширения удаления — повторять, пока множество не стабилизируется:
```
expanded := match.tiles
повторять:
  для каждого bomb в expanded: добавить 3×3
  для каждого rainbow в expanded: добавить все цвета match.color
до тех пор, пока expanded не перестало расти
```

После этого:
- Считаются очки: `(размер expanded) × 10 × multiplier`
- Все фишки в expanded удаляются
- На центральной позиции исходного матча, **если** длина матча 4/5+ И в матче **не было** активаций → создаётся спец-фишка (см. 2.3). Если активации были — обычная очистка без новых спец-фишек.

### 2.5 Цикл tilt с новыми правилами

Псевдокод:

```
tilt(direction):
  if isAnimating or isGameOver: return
  afterGravity = gravity(board, direction)
  if no movement: return

  isAnimating = true
  try:
    board = afterGravity
    render()
    await delay

    multiplier = 1
    while true:
      matches = findMatches(board)  // массив матч-объектов
      if matches empty: break

      // Для каждого матча: расширить через бомбы/радуги
      removeSet = empty set
      hadActivation = false
      for match in matches:
        for tile in match.tiles:
          if tile.special: hadActivation = true
        expanded = expandExplosions(match.tiles, board, match.color)
        removeSet |= expanded

      // Показать тоаст ×N если multiplier > 1
      if multiplier > 1:
        showComboToast(multiplier)
        triggerShake()

      // Триггер shake также если hadActivation (бомба/радуга сработала)
      if hadActivation:
        triggerShake()

      // Анимация удаления
      mark tiles for removal in DOM
      score += removeSet.size * 10 * multiplier
      multiplier++
      await delay
      remove tiles from board

      // Спавнить спец-фишки за матчи без активации
      for match in matches:
        if not match-had-active-special:
          if match.length == 4: place bomb at center
          if match.length >= 5: place rainbow at center

      board = gravity(board, direction)
      render()
      await delay

    // После цикла: спавн новых
    board = spawnTiles(board)
    render()
    await delay

    if isGameOver(board):
      showGameOver()
  finally:
    isAnimating = false
```

### 2.6 Видимые комбо (toast)

В DOM появляется элемент `<div id="combo-toast">` поверх поля. Когда `multiplier > 1`:

- Содержимое: `КОМБО ×{multiplier}!`
- CSS анимация: appear scale 0.5 → 1.2 → 1.0, opacity 0 → 1 → 0, длительность 600мс
- После окончания анимации скрывается (display: none или класс)
- Параллельные тоасты не нужны — на этот ход не возможны (волны идут последовательно)

### 2.7 Screen shake

CSS keyframe-анимация `shake` длительностью 200мс:
```
@keyframes shake {
  0%, 100% { transform: translate(0); }
  25% { transform: translate(-4px, 2px); }
  50% { transform: translate(3px, -2px); }
  75% { transform: translate(-2px, 3px); }
}
```

Применяется к `.game-area` или `.board` через временный класс `.shaking`. Триггерится из JS:
```
function triggerShake() {
  boardEl.classList.add('shaking');
  setTimeout(() => boardEl.classList.remove('shaking'), 200);
}
```

### 2.8 Визуальное оформление спец-фишек

**Бомба:** обычная фишка нужного цвета + дополнительная обводка (CSS box-shadow или border) с пульсацией. Например, белая обводка 2px, анимация `pulse` 1с loop (scale 1 → 1.05 → 1, opacity 1 → 0.7 → 1 для обводки).

CSS-класс `.tile.bomb` поверх `.tile.tile-N`.

**Радуга:** фигура без формы из 5 шаблона — у неё своя форма (например, круг) с фоном `linear-gradient` всех 5 цветов или `conic-gradient`. Лёгкое вращение (или статика — на усмотрение реализации, 2с rotate).

CSS-класс `.tile.rainbow`.

В `renderTiles()` после установки `tile-${type}` добавляется `.bomb` или `.rainbow` если есть `special`.

### 2.9 Очки за активации

Никаких отдельных бонусных очков за match-4 / match-5 / взрыв — формула остаётся `tiles_removed × 10 × multiplier`. Но теперь `tiles_removed` может быть значительно больше (взрыв 3×3 = до 9 фишек, активация радуги = ВСЕ фишки одного цвета на поле). Это автоматически даёт большой счёт за вау-моменты.

### 2.10 Game over с спец-фишками

`isGameOver` без изменений: проверяет, что ни один из 4 наклонов не двигает доску. Бомбы и радуги участвуют в проверке движения как обычные фишки.

Если у игрока на поле осталась только радуга и она зажата — это просто фишка, ничего особенного.

## 3. Файлы и границы

```
logic.js
  + findMatches: переписана для возврата массива матч-объектов + wildcard для радуги

  + (вспомогательные: isBomb, isRainbow)

  + tilt(): обновлён цикл (расширение, тоаст, shake, спавн спец-фишек)

  + #combo-toast (positioned absolute, hidden by default)

  + <div id="combo-toast" class="hidden"></div> поверх .board

  + Тесты для findMatches с радугой
  + Тесты для expandRemovalSet (бомба, радуга, цепочка бомб)
```

## 4. Что НЕ входит

- Бонусные очки за длинные матчи (формула × tiles × multiplier даёт это естественно)
- Звуки (как и раньше)
- Спец-фишки в стартовой раскладке
- Power-ups (молот, перемешать) — это отдельная история
- Превью следующих фишек
- Лимит на длину цепочки бомб — теоретически возможна цепная реакция через всё поле, что и должно быть вау

## 5. Критерии готовности

- [ ] Match-4 создаёт визуально различимую бомбу того же цвета
- [ ] Бомба, попавшая в любой матч, удаляет 3×3 вокруг себя
- [ ] Цепочка бомб (одна в радиусе другой) детонирует разом
- [ ] Match-5+ создаёт радужную фишку
- [ ] Радуга считается любым цветом в матче, при активации удаляет все фишки этого цвета
- [ ] Каскадные волны 2+ показывают тост "КОМБО ×N!"
- [ ] При комбо ≥2 и при активации спец-фишки поле трясётся
- [ ] Все 34+ существующих тестов проходят
- [ ] Новые юнит-тесты на findMatches с радугой, expandRemovalSet, цепочки бомб
- [ ] Manual playtest: 5 игр в headless-браузере не падают, бомбы/радуги срабатывают визуально
